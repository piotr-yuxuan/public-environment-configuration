#+TITLE: Security posture (macOS)
#+AUTHOR: caocoa
#+STARTUP: overview

Security hardening rationale and audit checklist for the macOS hosts
(aarch64-darwin and x86_64-darwin).  All measures described here are
implemented declaratively through nix-darwin (=hosts/macOS.nix=),
Home Manager (=home/macOS.nix=, =home/base.nix=), and the
post-activation script (=scripts/darwin-post-activation.sh=).

* Full disk encryption (FileVault)

The post-activation script checks FileVault status on every
=darwin-rebuild switch= and prints a warning to stderr if encryption
is off.  FileVault must be enabled manually through System Settings
because nix-darwin has no option to toggle it.

*Audit point:* run =fdesetup status= and verify "FileVault is On."

* System Integrity Protection (SIP)

The post-activation script verifies that SIP is enabled and warns if
it is not.  SIP is a boot-time setting managed by the Recovery OS, so
nix-darwin cannot enforce it, but the warning ensures it is not
accidentally left disabled after troubleshooting.

*Audit point:* run =csrutil status= and verify "enabled."

* Application Layer Firewall

=networking.applicationFirewall= in =hosts/macOS.nix= configures the
macOS Application Layer Firewall:

| Setting              | Value  | Purpose                              |
|----------------------+--------+--------------------------------------|
| =enable=             | =true= | Firewall enabled                     |
| =enableStealthMode=  | =true= | Stealth mode: drop unsolicited ICMP  |
| =allowSignedApp=     | =false= | Block incoming for signed downloads |

*Audit point:* open System Settings > Network > Firewall and confirm
it shows "Firewall is active."  Alternatively run
=/usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate=.

* ICMP redirect rejection

The post-activation script sets =net.inet.icmp.drop_redirect=1= on
every =darwin-rebuild switch=.  This prevents ICMP redirect injection
attacks on untrusted Wi-Fi networks (coffee shops, conferences).

*Audit point:* run =sysctl net.inet.icmp.drop_redirect= and verify
the value is 1.  This sysctl does not persist across reboots
natively; it is reapplied on the next =darwin-rebuild switch=.

* Gatekeeper verification

The post-activation script checks whether Gatekeeper assessments are
enabled and prints a warning to stderr if they are not.  Gatekeeper
itself is a macOS boot-time policy and cannot be toggled by
nix-darwin, so this acts as a detection measure.

*Audit point:* run =spctl --status= and verify "assessments enabled."

* Remote access lockdown

The post-activation script disables several remote access vectors on
every =darwin-rebuild switch=:

- *Remote Login (SSH server):* disabled via =systemsetup -setremotelogin off=.
- *Wake-on-LAN:* disabled via =pmset -a womp 0= to prevent powering
  on the machine from the network.
- *IR receiver:* disabled so the machine does not respond to infrared
  input.
- *Remote Apple Events:* disabled in =CustomUserPreferences=
  (=com.apple.AEServer=).

* Screen lock and idle timeout

=com.apple.screensaver= in =CustomUserPreferences= (=hosts/macOS.nix=)
configures automatic screen lock:

| Setting                | Value | Purpose                                     |
|------------------------+-------+---------------------------------------------|
| =askForPassword=       |     1 | Require password after screensaver or sleep  |
| =askForPasswordDelay=  |     0 | Immediately (no grace period)                |
| =idleTime=             |   900 | Screensaver activates after 15 min of idle   |

* Login window hardening

=system.defaults.loginwindow= in =hosts/macOS.nix= restricts the
login screen:

| Setting                | Value  | Purpose                                    |
|------------------------+--------+--------------------------------------------|
| =GuestEnabled=         | =false= | No guest sessions                         |
| =DisableConsoleAccess= | =true= | Prevent console (=>console=) login         |
| =SHOWFULLNAME=         | =true= | Username/password fields instead of avatars |

The post-activation script also enables the keyboard input menu on
the login screen so the layout tracks the last user.

* Siri and assistant disabled

=CustomUserPreferences= in =hosts/macOS.nix= disables Siri and the
assistant:

- =com.apple.Siri.StatusMenuVisible = false=
- =com.apple.Siri.UserHasDeclinedEnable = true=
- =com.apple.assistant.support."Assistant Enabled" = false=

* AirDrop and Bluetooth sharing

=CustomUserPreferences= in =hosts/macOS.nix= restricts wireless
sharing:

- *AirDrop:* set to contacts only (=com.apple.sharingd.DiscoverableMode=).
- *Bluetooth sharing:* disabled
  (=com.apple.Bluetooth.PrefKeyServicesEnabled = false=).

* Nix daemon trust model

=nix.settings.trusted-users= in =hosts/macOS.nix= is set to
=["root" "$USER"]= where =$USER= is resolved at evaluation time.
This scopes store substitution authority to the machine owner,
preventing future accounts from pushing arbitrary derivations.

*Audit point:* run =nix show-config | grep trusted-users= and verify
only the expected user is listed.

* Nix store signature verification

=nix.settings= in =hosts/macOS.nix= enforces binary cache integrity:

- =require-sigs = true=: every store path fetched from a cache must
  carry a valid cryptographic signature.
- =trusted-public-keys= is hardcoded to the official =cache.nixos.org=
  key so a rogue substituter cannot be silently accepted.

*Audit point:* run =nix show-config | grep require-sigs= and verify
it is =true=.

* sudo hardening (Touch ID)

=security.pam.services.sudo_local.touchIdAuth = true= in
=hosts/macOS.nix= enables Touch ID for sudo authentication.  This
provides biometric gating: shoulder surfing a password is not enough
to escalate privileges.

* Homebrew security environment

Home Manager (=home/macOS.nix=) sets the following environment
variables for Homebrew:

| Variable                        | Value            | Purpose                              |
|---------------------------------+------------------+--------------------------------------|
| =HOMEBREW_CASK_OPTS=            | =--require-sha=  | Require checksum on every cask       |
| =HOMEBREW_FORCE_BREWED_CURL=    | =1=              | Use Homebrew's curl (patched)        |
| =HOMEBREW_NO_ANALYTICS=         | =1=              | Disable telemetry                    |
| =HOMEBREW_NO_GITHUB_API=        | =1=              | No GitHub API calls                  |
| =HOMEBREW_NO_INSECURE_REDIRECT= | =1=              | Block HTTP redirects                 |
| =HOMEBREW_NO_ENV_HINTS=         | =1=              | Suppress hint banners                |

*Audit point:* run =env | grep HOMEBREW= and verify all variables are
set.

* SSH client hardening

Shared with C40C04 via =home/base.nix=.  Home Manager configures SSH
with:

- =IdentitiesOnly yes=: only the declared key is offered.
- =PasswordAuthentication no=: password fallback disabled.
- =HashKnownHosts yes=: host names in =known_hosts= are hashed.
- =StrictHostKeyChecking accept-new=: new hosts are accepted once,
  then pinned.
- =ForwardAgent no=: agent forwarding off by default.

*Audit point:* run =ssh -G github.com= and verify the settings match.

* Git supply chain verification

Shared with C40C04 via =home/base.nix=.  =transfer.fsckObjects=,
=fetch.fsckObjects=, and =receive.fsckObjects= are enabled.  The
cleartext =git://= protocol is blocked; only HTTPS and SSH are
allowed.

*Audit point:* run =git config --global --get transfer.fsckObjects=.

* Nix input staleness check

A launchd agent (=com.user.nix-staleness-check=) runs weekly on
Monday at 09:00.  It queries =nix flake metadata= and fires a macOS
notification if any input was last updated more than 14 days ago.

launchd automatically catches up missed triggers after sleep or
shutdown, so a laptop that was off over the weekend will run the check
on the next login.

*Audit point:* check the agent status with
=launchctl list | grep staleness=.

* Application sandboxing (TCC)

macOS Transparency, Consent, and Control (TCC) restricts per-app
access to sensitive directories (Desktop, Documents, Downloads).  For
side-loaded apps like VSCode that are not subject to the App Store
sandbox, TCC is the primary containment mechanism.

Recommended manual configuration in System Settings > Privacy &
Security > Files and Folders:

- Deny VSCode access to Desktop and Documents unless a project lives
  there.
- Deny access to Removable Volumes.
- Allow only the minimum necessary for the code directories you work
  in.

nix-darwin cannot manage TCC programmatically (it requires user
consent via the GUI).

*Audit point:* open System Settings > Privacy & Security > Files and
Folders and review the permissions granted to each application.

* Considerations for periodic audits

** Flake inputs

Run =nix flake metadata= in the repository directory and review the
dates of each input.  If any are more than two weeks old, run
=nix flake update= and rebuild.  The staleness agent should warn you
automatically.

** Firewall status

Run =/usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate=
and verify it reports enabled.

** Listening ports

Run =lsof -iTCP -sTCP:LISTEN -P -n= and verify nothing unexpected is
listening.

** FileVault status

Run =fdesetup status= and confirm encryption is active.

** Gatekeeper status

Run =spctl --status= and confirm assessments are enabled.

** SIP status

Run =csrutil status= and confirm System Integrity Protection is
enabled.

** ICMP redirect sysctl

Run =sysctl net.inet.icmp.drop_redirect= and verify the value is 1.
If it has been reset (e.g. after a macOS update), run
=darwin-rebuild switch= to reapply.

** OpenBSM audit logs

Review audit logs with
=sudo praudit -l /var/audit/current | tail -50=.

** Homebrew integrity

Run =brew doctor= and review any warnings about insecure
configurations or outdated formulae.
Run =brew update && brew outdated= to check for lagging packages.

** Nix store trust

Run =nix path-info --sigs /run/current-system= (if available) and
verify signatures come from trusted keys.

** TCC permissions

Open System Settings > Privacy & Security > Files and Folders.  Review
per-app access and revoke anything no longer needed.
