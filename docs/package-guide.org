* Build toolchain

** gcc
C/C++ compiler.  Used by Emacs to compile native modules (jinx, vterm).
#+begin_src sh
gcc -o hello hello.c
gcc -shared -fPIC -o module.so module.c   # build a shared library
#+end_src

** cmake
Cross-platform build system generator.
#+begin_src sh
cmake -B build -S .       # generate build files from CMakeLists.txt
cmake --build build       # compile the project
#+end_src

** pkg-config
Queries installed library flags for compilation.
#+begin_src sh
pkg-config --cflags --libs enchant-2   # print -I and -l flags for enchant
#+end_src

** enchant
Spell-checking meta-library.  Jinx (Emacs) compiles against it.  You
rarely call it directly. It is a compile-time dependency.

** meson / ninja
Modern build system pair.  Meson generates ninja files, ninja executes them.
#+begin_src sh
meson setup builddir
ninja -C builddir
#+end_src

* Languages & runtimes

** python3
#+begin_src sh
python3 -c "print('hello')"
python3 -m venv .venv && source .venv/bin/activate   # virtual environment
#+end_src

** nodejs
JavaScript runtime.  Also provides =npm=.
#+begin_src sh
node -e "console.log(42)"
npm init -y && npm install express
#+end_src

** rustup
Rust toolchain manager.  First run installs a default toolchain.
#+begin_src sh
rustup default stable          # install the stable toolchain
cargo new hello && cd hello && cargo run
#+end_src

* Clojure ecosystem

** clojure (C40C04 only)
Clojure CLI (=clj= / =clojure=) with tools.deps.
#+begin_src sh
clj                            # start a REPL
clj -X:deps tree               # show dependency tree
clojure -M:test                # run tests via alias in deps.edn
#+end_src

** babashka
Fast Clojure scripting with no JVM startup.
#+begin_src sh
bb -e '(+ 1 2)'                                # quick expression
bb -e '(-> (slurp "data.json") (json/parse-string))'
bb script.clj                                  # run a file
#+end_src

** clj-kondo
Clojure linter / static analysis.
#+begin_src sh
clj-kondo --lint src/            # lint a directory
clj-kondo --lint - <<< "(defn f [x] x x)"   # lint from stdin
#+end_src

** leiningen
Clojure project automation and build tool.
#+begin_src sh
lein new app my-project          # scaffold a new Leiningen project
lein repl                        # start a REPL
lein test                        # run tests
lein uberjar                     # build a standalone JAR
#+end_src

** neil
Manage =deps.edn= aliases and dependencies.
#+begin_src sh
neil dep add cheshire/cheshire   # add a dependency to deps.edn
neil dep upgrade                 # bump all deps to latest
neil new scratch my-project      # scaffold a new project
#+end_src

** Practicalli deps.edn
Not a package, but a flake input.  The community =deps.edn= from
=practicalli/clojure-deps-edn= is symlinked to
=~/.config/clojure/deps.edn=.  It gives you dozens of ready-made
aliases:
#+begin_src sh
clj -M:repl/rebel              # rich REPL with Rebel Readline
clj -M:test/runner              # run tests
clj -X:project/create           # create a new project
#+end_src
Run =clj -A:deps -Stree= to see everything available.

* Formatters

** black / isort / ruff
Python formatters and linter.
#+begin_src sh
black myfile.py                # reformat Python
isort myfile.py                # sort imports
ruff check src/                # lint Python (very fast)
ruff format src/               # format Python (alternative to black)
#+end_src

** yamlfmt / yamllint
#+begin_src sh
yamlfmt file.yaml              # auto-format YAML
yamllint -d relaxed file.yaml  # lint YAML
#+end_src

** shfmt / shellcheck
#+begin_src sh
shfmt -w script.sh             # format shell script in place
shellcheck script.sh           # lint shell script (catches quoting bugs, etc.)
#+end_src

** prettier
Formats HTML, CSS, JSON, Markdown, YAML, and more.
#+begin_src sh
prettier --write "**/*.json"
prettier --check .
#+end_src

* Testing

** bats
Shell test framework (Bash Automated Testing System). Tests live in
=tests/= and cover the =scripts/= activation snippets.
#+begin_src sh
bats tests/                                # run all test suites
bats tests/home-layout.bats               # run one file
bats --tap tests/                          # TAP-format output for CI
#+end_src

** ripgrep
Recursive regex search, faster than grep.
#+begin_src sh
rg "TODO" src/                 # search for TODO in src/
rg -t py "import os"           # search only Python files
rg -l "fixme"                  # list filenames only
#+end_src

** silver-searcher (ag)
Similar to ripgrep, older but still fast.
#+begin_src sh
ag "pattern" .
ag --clojure "defn"            # search only Clojure files
#+end_src

** fzf
Fuzzy finder.  Any list piped in becomes an interactive selector.
Configured in =home/base.nix= with =fd= as the file-listing backend so
=.gitignore= is respected and hidden files are included.

*** Shell key bindings (always available at the prompt)
| Key     | Action                                            |
|---------+---------------------------------------------------|
| =C-t=   | Paste selected file(s) onto the command line      |
| =C-r=   | Fuzzy-search shell history; =C-r= again for chronological order |
| =M-c=   | =cd= into a selected subdirectory                 |

*** Inside the fzf popup
| Key       | Action                          |
|-----------+---------------------------------|
| =C-a=     | Toggle-select all matches       |
| =C-/=     | Toggle preview pane             |
| =C-u=     | Scroll preview half-page up     |
| =C-d=     | Scroll preview half-page down   |
| =Alt-↑=   | Jump to top of preview          |
| =Alt-↓=   | Jump to bottom of preview       |
| =Tab=     | Mark/unmark one item            |

*** Worked examples
#+begin_src sh
# Open a file in $EDITOR, previewing it with bat before committing.
# Press C-/ to show/hide the preview pane.
$EDITOR $(fzf)

# Kill one or more processes interactively (multi-select with Tab).
kill -9 $(ps aux | fzf --multi | awk '{print $2}')

# Check out a git branch.
git checkout $(git branch -a | fzf)

# Replay a recent git commit as a patch to the working tree.
git show $(git log --oneline | fzf | awk '{print $1}') | git apply

# Fuzzy-grep then open the matching file at the right line.
rg --line-number '' | fzf --delimiter ':' --nth 1,3.. \
    --preview 'bat --color=always --highlight-line {2} {1}' \
  | awk -F: '{print "+"$2, $1}' | xargs $EDITOR

# Copy a selected file path to the clipboard (Wayland).
fzf | wl-copy

# fzf also works inside scripts: read one line from any list.
selected=$(printf '%s\n' alpha beta gamma | fzf)
#+end_src

** zoxide
Smarter =cd= that learns your most-visited directories.
#+begin_src sh
z nixos                        # jump to most-used dir matching "nixos"
zi                             # interactive selection
#+end_src

** fd
User-friendly =find= replacement. Also drives fzf file and directory search.
#+begin_src sh
fd "\.nix$"                    # find all .nix files
fd -e py -x black {}           # find Python files and format them
#+end_src

* CLI utilities

** fortune
Print a random quotation. Runs once at login as part of the shell greeting.
#+begin_src sh
fortune                         # random quote
fortune -s                      # short quotes only
#+end_src

** macchina
System information fetcher (modern replacement for archey/neofetch).
Runs once at login as part of the shell greeting.
#+begin_src sh
macchina                        # show system info
macchina -t Hydrogen            # use a specific theme
#+end_src

** jq
JSON processor.
#+begin_src sh
curl -s https://api.github.com/users/octocat | jq '.name'
echo '[1,2,3]' | jq 'map(. * 2)'
cat data.json | jq -r '.[] | .id'
#+end_src

** yq-go
YAML processor, same idea as jq.
#+begin_src sh
yq '.services.web.image' docker-compose.yml
yq -i '.version = "2"' config.yaml          # edit in place
#+end_src

** fx
Interactive JSON viewer (TUI).
#+begin_src sh
curl -s https://api.github.com/repos/NixOS/nixpkgs | fx
cat large.json | fx '.items[0]'
#+end_src

** curl
HTTP client.
#+begin_src sh
curl -sL https://example.com
curl -X POST -H "Content-Type: application/json" -d '{"key":"val"}' https://api.example.com/data
#+end_src

** tokei
Count lines of code by language.
#+begin_src sh
tokei .                        # summary of the whole repo
tokei src/ --sort code         # sorted by lines of code
#+end_src

** tree
Directory listing as a tree.
#+begin_src sh
tree -L 2                      # depth-limited to 2
tree -I node_modules           # exclude a directory
#+end_src

** enca
Detect and convert file encodings.
#+begin_src sh
enca -L fr fichier.txt         # detect encoding (French heuristics)
enca -x UTF-8 fichier.txt     # convert to UTF-8 in place
#+end_src

** qrencode
Generate QR codes.
#+begin_src sh
qrencode -o qr.png "https://example.com"
echo "wifi secret" | qrencode -t UTF8        # print QR in terminal
#+end_src

** duckdb
In-process analytical SQL engine like SQLite for analytics.
#+begin_src sh
duckdb                                       # interactive REPL
duckdb -c "SELECT count(*) FROM 'data.csv'"
duckdb -c "SELECT * FROM read_parquet('file.parquet') LIMIT 5"
#+end_src

** poppler-utils
PDF text extraction tools.
#+begin_src sh
pdftotext document.pdf -        # extract text to stdout
pdfinfo document.pdf            # show metadata (pages, size, etc.)
#+end_src

** asciinema
Record and share terminal sessions.
#+begin_src sh
asciinema rec demo.cast         # start recording
asciinema play demo.cast        # replay locally
asciinema upload demo.cast      # share on asciinema.org
#+end_src

** bmon
Real-time network bandwidth monitor (TUI).
#+begin_src sh
bmon                            # launch the monitor
bmon -p wlp1s0                  # monitor a specific interface
#+end_src

** yt-dlp
Download videos/audio from YouTube and hundreds of other sites.
#+begin_src sh
yt-dlp "https://www.youtube.com/watch?v=dQw4w9WgXcQ"
yt-dlp -x --audio-format mp3 URL              # extract audio as MP3
yt-dlp -f "bestvideo+bestaudio" URL            # best quality
yt-dlp --list-formats URL                      # show available formats
#+end_src

** ffmpeg
Swiss-army knife for audio/video processing.
#+begin_src sh
ffmpeg -i input.mp4 output.mkv                 # convert format
ffmpeg -i video.mp4 -ss 00:01:00 -t 30 clip.mp4   # extract 30s clip
ffmpeg -i input.wav -codec:a libopus output.opus   # transcode audio
ffmpeg -i video.mp4 -vn -acodec copy audio.m4a     # strip video, keep audio
#+end_src

* IaC / Cloud

** awscli2
Official AWS CLI.
#+begin_src sh
aws sts get-caller-identity                    # who am I?
aws s3 ls s3://my-bucket/                      # list bucket contents
aws cloudformation describe-stacks --stack-name my-stack
#+end_src

** aws-vault
Securely store and access AWS credentials (uses OS keychain).
#+begin_src sh
aws-vault add work                             # store credentials for "work" profile
aws-vault exec work -- aws s3 ls               # run a command with temporary creds
aws-vault login work                           # open AWS console in browser
#+end_src

** granted
Fast AWS role switching.
#+begin_src sh
assume work-admin                              # switch to a role (interactive)
assume --export work-admin                     # export creds as env vars
#+end_src

** ssm-session-manager-plugin
Start SSH-like sessions to EC2 instances via AWS SSM (no open ports).
#+begin_src sh
aws ssm start-session --target i-0123456789abcdef0
#+end_src

** sops
Encrypted secrets management (works with age, gpg, AWS KMS).
#+begin_src sh
sops secrets.yaml                              # edit encrypted file (decrypts in $EDITOR)
sops -d secrets.yaml                           # decrypt to stdout
sops -e -i secrets.yaml                        # encrypt a plaintext file in-place
sops --age $(cat ~/.config/sops/age/keys.txt | grep public) -e plain.yaml > enc.yaml
#+end_src

** age
Modern file encryption, simpler than GPG.
#+begin_src sh
age-keygen -o key.txt                          # generate a keypair
age -r age1... -o secret.enc secret.txt        # encrypt to a recipient
age -d -i key.txt secret.enc                   # decrypt with your key
#+end_src

** tenv
Terraform version manager (like tfenv but multi-tool).
#+begin_src sh
tenv tf install 1.9.0                          # install a Terraform version
tenv tf use 1.9.0                              # switch to it
tenv tofu install latest                       # also manages OpenTofu
#+end_src

** terraform-ls / terraform-lsp
Language servers for Terraform (=.tf= files).  Used by Eglot / VS Code.

** terraform-landscape
Reformats =terraform plan= output for readability.
#+begin_src sh
terraform plan | landscape
#+end_src

** podman
Rootless container engine (Docker-compatible).
#+begin_src sh
podman run --rm -it alpine sh                  # run a throwaway container
podman build -t myapp .                        # build from Dockerfile
podman ps                                      # list running containers
#+end_src

* CloudFormation

** python3Packages.cfn-lint
CloudFormation template linter.
#+begin_src sh
cfn-lint template.yaml                         # lint a template
cfn-lint "templates/**/*.yaml"                 # lint all templates in a directory
cfn-lint -f json template.yaml                 # JSON output for CI
#+end_src

** rain
CloudFormation deploy / diff CLI with much better UX than =aws cloudformation=.
#+begin_src sh
rain deploy template.yaml my-stack             # deploy (waits, streams events)
rain diff template.yaml my-stack               # show what would change
rain cat my-stack                              # print current template from AWS
rain logs my-stack                             # show recent CloudWatch logs
rain fmt template.yaml                         # format a CFN template
#+end_src

* Load testing

** k6
Modern load testing with tests written in JavaScript.
#+begin_src sh
# Create a test script:
cat > test.js <<'EOF'
import http from 'k6/http';
export default function () {
  http.get('https://test-api.example.com/');
}
EOF
k6 run test.js                                 # default: 1 VU, 1 iteration
k6 run --vus 50 --duration 30s test.js         # 50 virtual users for 30s
#+end_src

** vegeta
HTTP load testing that is pipe-based and composable.
#+begin_src sh
echo "GET https://example.com/" | vegeta attack -duration=10s | vegeta report
echo "GET https://example.com/" | vegeta attack -rate=100/s | vegeta plot > plot.html
#+end_src

** hey
Simple HTTP load generator: one command, instant results.
#+begin_src sh
hey -n 1000 -c 50 https://example.com/        # 1000 requests, 50 concurrent
hey -m POST -d '{"key":"val"}' -T "application/json" https://api.example.com/
#+end_src

* LaTeX

** texliveFull
Full TeX Live distribution.  Includes XeLaTeX, LuaLaTeX, pdfLaTeX,
latexmk, and all CTAN packages.
#+begin_src sh
xelatex document.tex           # compile with XeLaTeX (for system fonts)
lualatex document.tex          # compile with LuaLaTeX (for advanced font features)
latexmk -xelatex document.tex # auto-build with latexmk (watches dependencies)
latexmk -c                     # clean auxiliary files
#+end_src

** tectonic
Self-contained TeX engine that downloads packages on demand.
#+begin_src sh
tectonic document.tex          # compile (fetches missing packages automatically)
tectonic -X build              # build a Tectonic.toml project
#+end_src

** texlab (macOS only)
LaTeX language server.  Used by Eglot / VS Code for completion,

** vlc (C40C04 only)
Media player that handles virtually any audio/video format.
#+begin_src sh

cvlc --play-and-exit file.mp3  # headless playback
#+end_src

VLC uses VA-API hardware decoding on the AMD Radeon 780M (Phoenix)
iGPU via the =radeonsi= Mesa driver.  The VCN v4.0 unified decode
ring can time out under heavy HEVC/AV1 workloads; the kernel resets
the ring successfully, but Mesa treats the context reset as fatal and
calls =abort()=, crashing VLC.  Three =amdgpu= kernel parameters in
=hosts/C40C04.nix= mitigate this:

| Parameter                  | Value                    | Purpose                                              |
|----------------------------+--------------------------+------------------------------------------------------|
| =timeout_fatal_disable=    | =1=                      | Non-fatal ring resets (Mesa no longer aborts)         |
| =lockup_timeout=           | =10000,10000,10000,10000= | 10 s per ring (GFX, Compute, SDMA, Video) vs 2 s default |
| =gpu_recovery=             | =1=                      | Explicit ring reset (default is auto)                |

To verify after reboot:
#+begin_src sh
cat /sys/module/amdgpu/parameters/timeout_fatal_disable   # expected: Y
cat /sys/module/amdgpu/parameters/lockup_timeout           # expected: 10000,10000,10000,10000
cat /sys/module/amdgpu/parameters/gpu_recovery             # expected: 1
#+end_src

If VCN timeouts remain very frequent, consider updating
=linux-firmware= (newer VCN v4.0 firmware), trying a newer kernel, or
falling back to software decode (=avcodec-hw=none= in =vlcrc=).

** nh
Nix CLI helper that wraps =nixos-rebuild=, =darwin-rebuild=, and
=home-manager= with a nicer interface: a build tree via
=nix-output-monitor=, a coloured diff of what is about to change via
=dix=, and an interactive confirmation prompt before activation.

Installed fleet-wide via =home/base.nix=.  On C40C04 the NixOS module
(=programs.nh= in =hosts/C40C04.nix=) also enables a weekly =nh clean=
systemd timer that prunes old generations (keeps 5, anything newer than

schedule (Sunday 03:00) via =hosts/macOS.nix=.

The =NH_FLAKE= session variable is intentionally unset: the flake path
is user-specific and must be supplied at the call site or via a personal
shell alias.  On C40C04, =NH_OS_FLAKE= is set automatically by the NixOS
module (=programs.nh.flake=) and takes precedence over =NH_FLAKE= for
=nh os= commands.

*** Seeing a human-friendly diff before switching

This is the key workflow: build the new configuration, show you exactly
what derivations and closures are added, changed, or removed, then ask
for confirmation before touching the running system.

On NixOS (C40C04):
#+begin_src sh
nh os switch          # reads NH_OS_FLAKE; auto-detects hostname
nh os switch -H C40C04   # explicit host
#+end_src

On macOS:
#+begin_src sh
nh darwin switch          # reads NH_FLAKE; auto-detects hostname
#+end_src

For Home Manager on any platform:
#+begin_src sh
nh home switch          # reads NH_FLAKE
#+end_src

In all three cases =nh= will:
1. Evaluate the flake and build the new system closure (output routed
   through =nix-output-monitor= for a readable build tree).
2. Display a =dix= diff: a compact, colour-coded summary of every
   package added, removed, or version-bumped between the current and
   new generation.
3. Ask =Apply? [y/N]= before running the activation script.

To preview without any risk of activation, use =--dry=:
#+begin_src sh
nh os switch --dry          # build + diff, never activate
nh darwin switch --dry
nh home switch --dry
#+end_src

** USB-C monitor hotplug stabilisation
Workaround for a known Infineon PD controller firmware race on
Framework AMD laptops that causes the Iiyama PL3494WQ's built-in
Realtek USB hub to enter a rapid connect/disconnect cycle.  No
upstream kernel patch or firmware update exists (as of March 2026).
Configured in =hosts/C40C04.nix=.

References:
- https://community.frame.work/t/tracking-usb-c-error-on-boot/52012
- https://community.frame.work/t/display-connecting-infinitely-via-usb-c/68717

Levers:
- udev (=config/udev/99-iiyama-usb-hub.rules=) pins the Realtek hub
  and xHCI controller to active power.
- Watchdog service (=usbc-monitor-stabilize=) detects rapid
  disconnects and reloads =ucsi_acpi= + rebinds the xHCI controller
  to break the cycle.
#+begin_src sh
systemctl status usbc-monitor-stabilize              # watchdog running
cat /sys/bus/pci/devices/0000:c3:00.3/power/control   # should be on
cat /sys/bus/pci/devices/0000:c3:00.3/d3cold_allowed   # should be 0
#+end_src
