#+title: Tips and recipes
#+author: piotr-yuxuan

A collection of shell snippets, utilities, and hard-won notes
accumulated over several years of environment tinkering first on macOS
with raw dotfiles, now across NixOS and macOS with Nix.

Some of these are timeless (Z-shell history options, GPG terminal
setup); others are macOS-specific conveniences. Defunct tools
(oh-my-zsh, zplug, jenv) are kept in the [[*Graveyard][Graveyard]] for posterity.

* Z-shell                                                              :tty:

** History configuration

Large, deduplicated, shared history is one of the highest-leverage
shell tweaks. These options are set in =config/after.zshrc= via Home
Manager now, but the reasoning still applies.

#+begin_src zsh
export HISTSIZE=10000000
export HISTFILESIZE=100000000
export HISTTIMEFORMAT="%d/%m/%y %T "

setopt EXTENDED_HISTORY          # Write ":start:elapsed;command" format.
setopt INC_APPEND_HISTORY        # Write immediately, not on shell exit.
setopt SHARE_HISTORY             # Share history between all sessions.
setopt HIST_EXPIRE_DUPS_FIRST    # Expire duplicates first when trimming.
setopt HIST_IGNORE_DUPS          # Don't record consecutive duplicates.
setopt HIST_IGNORE_ALL_DUPS      # Delete old entry if new is a duplicate.
setopt HIST_FIND_NO_DUPS         # Don't display previously found lines.
setopt HIST_IGNORE_SPACE         # Don't record entries starting with a space.
setopt HIST_SAVE_NO_DUPS         # Don't write duplicate entries to the file.
setopt HIST_REDUCE_BLANKS        # Remove superfluous blanks before recording.
#+end_src

** Don't compile Z-shell startup files

It looks tempting to =zcompile= startup files to boost start-up time.
However, whenever a script silently appends to a Z-shell configuration
file (e.g. an installer adding something to =$PATH=), the source file
gets a newer timestamp than the compiled file and takes precedence.
Keep it simple: don't compile startup files so you can modify them
without thinking about it.

* SSH configuration

Express only connection-specific settings here. Git SSH keys are
handled through gitconfig's =includeIf= and per-host =IdentityFile= lines.

A representative =~/.ssh/config= pattern:

#+begin_src ssh-config
Host *
  UseKeychain           yes    # macOS keychain integration
  AddKeysToAgent        yes

Host github.com
  HostName              github.com
  User                  git
  IdentityFile          ~/.ssh/id_ed25519
  ServerAliveInterval   20
  ForwardAgent          no

Host gitlab.com
  HostName              gitlab.com
  User                  git
  IdentityFile          ~/.ssh/gitlab_id_ed25519
  ServerAliveInterval   20
  ForwardAgent          no
#+end_src

On NixOS, =programs.ssh= in Home Manager generates this file
declaratively. On macOS, the file is kept in =config/= and symlinked.

** Change directory to frontmost Finder window

#+begin_src zsh
pfd () {
    currFolderPath=$( /usr/bin/osascript <<EOT
        tell application "Finder"
            try
                set currFolder to (folder of the front window as alias)
            on error
                set currFolder to (path to desktop folder as alias)
            end try
            POSIX path of currFolder
        end tell
EOT
    )
    cd "$currFolderPath"
}
#+end_src

* fzf, command-line fuzzy finder

https://github.com/junegunn/fzf

#+begin_src zsh
[ -f ~/.fzf.zsh ] && source ~/.fzf.zsh
#+end_src

Key bindings:
- =C-t= paste selected files/directories onto the command line
- =C-r= paste selected command from history (=C-r= again for chronological
  order)
- =M-c= cd into selected directory

On both platforms, =programs.fzf.enable = true= in Home Manager sets
this up automatically.

* Query structured data

** Grep JSON with gron

#+begin_src zsh
# brew install gron  /  nix-shell -p gron
gron "https://api.github.com/repos/tomnomnom/gron/commits?per_page=1" \
  | fgrep "commit.author" \
  | gron --ungron
#+end_src

=gron= is like =jq= but less powerful and easier to use: its purpose is
finding the path to a value in a deeply nested JSON blob when you
don't already know the structure.

* Reduce the size of a music library

Extract audio from video containers:

#+begin_src zsh
# Single file
if ffmpeg -i a.mkv -vn -acodec copy a.ogg; then
    rm a.mkv
else
    echo "Failed for a.mkv"
fi

# All MKV files recursively
find . -type f -name "*.mkv" -exec bash -c '
  FILE="$1"
  if ffmpeg -i "$FILE" -vn -acodec copy "$FILE.ogg"; then
      rm "$FILE"
  else
      echo "Failed for $FILE"
  fi
' _ '{}' \;
#+end_src

Ogg is a container format that can embed audio in various codecs
without re-encoding, so quality stays the same.  Typical size
reduction from MKV: ~66%.

** Don't start Emacs on login (macOS)

On macOS, the plist system doesn't support ordering (unlike systemd).
If Emacs starts before =pinentry= is available, Magit GPG-signed commits
fail. Simpler to let Emacs start on first use: oh-my-zsh's =emacs=
plugin or a manual =emacsclient= wrapper handle this.

On NixOS, Emacs runs as a systemd user service with proper dependency
ordering.
